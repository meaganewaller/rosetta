# Learn

Extract reusable patterns from the current session and save them as skills.

## When to run

After solving a non-trivial problem — an error resolution, a non-obvious debugging technique, a library quirk, a codebase convention, or an integration pattern that will save time in future sessions.

Don't extract trivial fixes (typos, syntax errors) or one-time issues (specific outages). One pattern per skill file.

## What to do

1. Review the session for extractable patterns
2. Draft a skill file at `~/.claude/skills/learned/<pattern-name>.md` covering: the problem it solves, the solution or technique, an example if applicable, and when to activate it
3. Show the draft to the user and confirm before saving
